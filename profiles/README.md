# Hardware Calibration for v0.1 Experiment Matrix

This directory contains tools for calibrating and managing hardware profiles for the v0.1 experiment matrix.

## Overview

The v0.1 experiment matrix requires a **single fixed hardware profile** for all experiments on a Vast 3×3090 machine. This profile is sized against the worst-case regime:

- **Architecture**: FlowMidblock (A3) — most complex
- **Loss**: End + Traj + KL + CE — maximum target computation
- **Data**: Mix C — largest corpus
- **seq_len**: 1024 — fixed for all v0.1 experiments

## Files

- `v0_1_3090_profile.json` — Hardware profile artifact (generated by calibration)
- `calibration_worst_case.yaml` — Worst-case config for calibration runs
- `../scripts/calibrate_hardware.py` — Calibration script
- `../src/utils/hardware_profile.py` — Profile loader module

## Hardware Profile Format

```json
{
  "hardware": "NVIDIA RTX 3090 24GB",
  "seq_len": 1024,
  "microbatch_size": 2,
  "gradient_accumulation": 8,
  "effective_batch_size": 16,
  "precision": "bf16-mixed",
  "gradient_checkpointing": true,
  "peak_vram_gb": 21.5,
  "tokens_per_sec": 420,
  "calibrated_on": "FlowMidblock_EndTrajKLCe_MixC",
  "calibration_timestamp": "2026-04-03T00:00:00Z"
}
```

## Usage

### Running Calibration

On a machine with a 3090 GPU:

```bash
python scripts/calibrate_hardware.py \
    --config configs/calibration_worst_case.yaml \
    --target-batch-size 16 \
    --vram-limit 22.0 \
    --output profiles/v0_1_3090_profile.json
```

This will:
1. Sweep microbatch sizes from 1 upward
2. Monitor peak VRAM usage
3. Add gradient accumulation to reach effective batch size
4. Record tokens/sec for each stable configuration
5. Write the optimal profile to `profiles/v0_1_3090_profile.json`

### Using the Profile in Experiments

```python
from src.utils.hardware_profile import load_v0_1_profile, apply_hardware_profile_to_config

# Load the profile
profile = load_v0_1_profile()

# Apply to config
config = load_config("configs/experiment.yaml")
updated_config = apply_hardware_profile_to_config(config, profile)
```

### Manual Profile Updates

If you need to manually adjust the profile (e.g., for a different GPU):

```json
{
  "hardware": "NVIDIA RTX 4090 24GB",
  "seq_len": 1024,
  "microbatch_size": 4,
  "gradient_accumulation": 4,
  "effective_batch_size": 16,
  "precision": "bf16-mixed",
  "gradient_checkpointing": true,
  "peak_vram_gb": 20.0,
  "tokens_per_sec": 850,
  "calibrated_on": "FlowMidblock_EndTrajKLCe_MixC",
  "calibration_timestamp": "2026-04-03T12:00:00Z"
}
```

## Calibration Strategy

1. **Phase 1**: Find max stable microbatch with grad_accum=1
   - Test microbatch=1, 2, 3, ... until OOM or >22GB
   - Record peak VRAM and tokens/sec for each

2. **Phase 2**: Add gradient accumulation
   - Calculate: `grad_accum = target_batch_size / microbatch_size`
   - Verify final effective batch size

3. **Phase 3**: Verify final profile
   - Run 5 steps with final configuration
   - Confirm peak VRAM < 22GB
   - Record final tokens/sec

## Constraints

- **VRAM limit**: 22GB (leaving headroom on 24GB cards)
- **Target effective batch size**: 16 (configurable)
- **Precision**: bf16-mixed (mandatory for 3090)
- **Gradient checkpointing**: Required for large models
- **seq_len**: 1024 (fixed for all v0.1 experiments)

## Integration with Queue Runner

The queue runner automatically loads the hardware profile and applies it to all experiment configs:

```python
from src.utils.hardware_profile import load_v0_1_profile

profile = load_v0_1_profile()
# profile is applied to each config before training starts
```
